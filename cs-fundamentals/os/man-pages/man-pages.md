# What are man pages??

Man pages (short for manual pages) are the traditional, built-in documentation on UNIX and UNIX-like systems (Linux, macOS, BSD, etc.). They document commands, shell built-ins, library functions, system calls, configuration files, and similar topics.

From the excerpt you shared:

- They predate the web — They were the standard way to document this stuff before online docs and browsers were common.

- How you use them — The book tells you to “read the manual pages” when it points you at a specific system call or library call. You open them with the man command (e.g. man fork, man bash).

- Why they matter for systems programming — They spell out return values, error conditions, flags, and behavior details that are easy to get wrong if you only guess from code examples.

“**RTFM**” — Colleagues saying “Read The F Man pages” is shorthand for: check the man page first; the answer is often there and saves you from asking redundant questions.
